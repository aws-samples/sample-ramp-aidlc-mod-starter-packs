## Quick Start

### 1. Install Dependencies

```bash
cd {{AGENT_NAME}}
python -m venv .venv
source .venv/bin/activate  # On Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

### 2. Configure Environment

```bash
cp .env.example .env
# Edit .env with your configuration
```

Required environment variables:
- `AWS_REGION`: AWS region (default: us-east-1)
- `MODEL_ID`: Bedrock model ID
- `TENANT_ID`: Your tenant identifier

### 3. Deploy Infrastructure

```bash
cd terraform
terraform init
terraform plan -var="tenant_id=your-tenant-id" -var="environment=dev"
terraform apply
```

### 4. Test Locally

```bash
# Set environment variables
export AWS_REGION=us-east-1
export MODEL_ID=us.anthropic.claude-sonnet-4-5-20250929-v1:0
export TENANT_ID=test-tenant

# Run the agent locally
python -m src.agent
```

### 5. Test the API

```bash
# Get the API endpoint from Terraform output
API_ENDPOINT=$(terraform output -raw api_endpoint)

# Send a test request
curl -X POST $API_ENDPOINT \
  -H "Content-Type: application/json" \
  -H "x-tenant-id: your-tenant-id" \
  -d '{"task": "Hello, can you help me?"}'
```

## Troubleshooting

### Common Issues

**Error: Missing tenant_id**
- Ensure `x-tenant-id` header is included in requests

**Error: Access Denied to Bedrock**
- Verify IAM role has Bedrock permissions
- Check model access is enabled in Bedrock console

**Error: Lambda Timeout**
- Increase `lambda_timeout` variable
- Consider moving to ECS for longer tasks

### Logs

```bash
# View recent logs
aws logs tail /aws/lambda/{{AGENT_NAME}} --follow

# Search for errors
aws logs filter-log-events \
  --log-group-name /aws/lambda/{{AGENT_NAME}} \
  --filter-pattern "ERROR"
```
